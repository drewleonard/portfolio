Drew Navin LeonardProjects
Mass Killings visualizes the risks of state-led mass killings around the world. I built it for the Early Warning Project during Summer 2017.
Link to project: https://goo.gl/U7dZGj
Link to GitHub repository: https://goo.gl/zCqpBH
Details
This project leverages JavaScript (D3.js, jQuery.js), HTML5, and CSS3 to plot risks of state-led mass killings for countries with populations above 500,000. Users can use the left bar chart to view each country's percentage risk, which are ordered by magnitude. The right bar chart displays the variables that comprise risk assessments and vary by country.
Data for risk assessments and their component parts come from the Early Warning Project, which compiled data from various open-source datsets and performed statistical forecasts. Risk assessments are averages of forecasts from three models: Bad Regime, Elite Threat, and Random Forest. These models represents different ideas about the origins of mass atrocities, and are developed from historical data that compare cases where state-led mass killings occurred to ones where they didnâ€™t. Further details on the data are provided in this project's GitHub repository.
The project is deployed with Surge. Previously, I used Mike Bostock's bl.ocks.org and gistup software to host the website. The project's color scheme utilizes the Solarized color palette.
Motivation
I created this project for the Early Warning Project, which is an initiative to warn governments, advocacy groups, and at-risk societies of mass atrocities early. In visualizing each country's risk of a state-led mass killings and that forecast's component parts, my goal is to provide context necessary to understand how the Early Warning Project generates each forceast and how each variable contributes. Prior to this visualization, audiences interested in learning about the component parts of risk assessments could read about the models and their variables, but would have to sift through plain text files containing comma-separated-values to view the data behind each assessment. This project provides a centralized and clear format to view those component parts.
The project's design draws inspiration from no particular visualization or idea. I built it while I was learning to use JavaScript (and D3.js), HTML5, and CSS3, and I did not have a clear design when I started. While I assumed this project was a problem of data visualization, it became clear quickly that it was one of web design also. Accordingly, I built the project around two or three times in total to accomodate necessary changes to host this in its own website in a meaningful and elegant way. While two bar charts are not the most exciting method to visualize data, I believe the project fulfills its educational mission well.
